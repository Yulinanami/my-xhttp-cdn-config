# XHTTP + CDN 搭建指南 (四)：ECH 配置

本文档是在 [3.xpadding配置.md](./3.xpadding配置.md) 的基础上，补充 **ECH** 相关配置。

---

## 1. 作用范围

ECH 建议加到 **CDN-TLS** 相关节点：

- 上行 `xhttp+TLS+CDN`
- `xhttp+TLS` 双向 CDN
- `download-settings` 里的下行 `xhttp+TLS+CDN`

服务端 Xray / Nginx 不需要因为客户端 ECH 做额外修改。

---

## 2. Xray / v2rayN 客户端

使用 Cloudflare 公共 ECH 名称：

```text
cloudflare-ech.com+https://223.5.5.5/dns-query
```

### 2.1 普通 CDN-TLS 节点

在 EchConfigList 里加入：

```json
"echConfigList": "cloudflare-ech.com+https://223.5.5.5/dns-query"
```

对应 VLESS 分享链接参数为：

```text
ech=cloudflare-ech.com%2Bhttps%3A%2F%2F223.5.5.5%2Fdns-query
```

### 2.2 `downloadSettings` 里的 CDN-TLS

在 `downloadSettings.tlsSettings` 里加入：

```json
"tlsSettings": {
  "serverName": "YOUR_CDN_DOMAIN",
  "allowInsecure": false,
  "alpn": [
    "h2"
  ],
  "fingerprint": "chrome",
  "echConfigList": "cloudflare-ech.com+https://223.5.5.5/dns-query"
}
```

## 3. Mihomo 客户端

Mihomo 使用 `ech-opts`。

### 3.1 普通 CDN-TLS 节点

放在节点顶层，和 `servername`、`client-fingerprint`、`xhttp-opts` 同级：

```yaml
servername: "YOUR_CDN_DOMAIN"
client-fingerprint: chrome
ech-opts:
  enable: true
  query-server-name: "cloudflare-ech.com"
xhttp-opts:
  host: "YOUR_CDN_DOMAIN"
  path: "YOUR_PATH"
```

### 3.2 `download-settings` 里的 CDN-TLS

放在 `download-settings` 里，和 `tls`、`servername`、`client-fingerprint` 同级：

```yaml
download-settings:
  server: "YOUR_CDN_DOMAIN"
  port: 443
  tls: true
  alpn:
    - h2
  servername: "YOUR_CDN_DOMAIN"
  client-fingerprint: chrome
  ech-opts:
    enable: true
    query-server-name: "cloudflare-ech.com"
```
