# XHTTP + CDN 搭建指南 (一)：环境准备

本文档旨在记录并快速复现基于 `Xray-core` 的 xhttp 传输出站方案（支持 Reality 直连与 CDN 上下行分离）。

**参考资料**
1. 文档1：https://github.com/XTLS/Xray-core/discussions/4118
2. 文档2：https://jollyroger.top/sites/361.html

---

## 0. 准备工作 (材料清单)
在开始敲代码之前，请确保你已经准备好以下核心材料：
1. **一台的 VPS**
2. **两个域名**：
   * `cdn.yourdomain2.com` -> 用于走 CDN 的流量
   * `reality.yourdomain1.com` -> 用于 Reality 直连伪装
3. **一个伪装路径**：可以随便写，例如 `/d3f2a994`

---

## 1. Cloudflare 设置 

登录 Cloudflare 控制台，按顺序完成以下设置：

### 1.1 DNS 解析设置
* `cdn` 域名记录：**开启** 代理状态（点亮小黄云 橙色）。
* `reality` 域名记录：**关闭** 代理状态（仅限 DNS 灰色）。

### 1.2 SSL/TLS 加密模式
* 必须将加密模式设置为 **完全（严格） / Full (Strict)**。

### 1.3 开启 gRPC 转发
* 进入 **网络 (Network)**，向下滚动找到 **gRPC** 选项，并将其开关 **打开**。

## 2. 安装必要组件与 Xray

连接到终端并切换到 root 权限。

```bash
# 切换到root用户
sudo -i

# 更新（有询问的话回车就好了）
apt update && apt upgrade

# 安装必要组件
apt install -y curl sudo

# 安装 xray
bash -c "$(curl -L https://github.com/XTLS/Xray-install/raw/main/install-release.sh)" @ install -u root
```

---

## 3. 安装 Acme.sh 申请双域名证书

```bash
# 安装 acme 环境
apt install socat cron -y && curl https://get.acme.sh | sh && ln -s /root/.acme.sh/acme.sh /usr/local/bin/acme.sh && ln -s /root/.acme.sh/acme.sh /usr/local/bin/acme.sh

# 切换 CA 为 Let's Encrypt
acme.sh --set-default-ca --server letsencrypt

# 合并申请双域名证书 (替换为你自己的域名)
acme.sh --issue -d reality.yourdomain1.com -d cdn.yourdomain2.com --standalone

# 将证书安装到系统目录供后续 Nginx 读取 (注意 -d 后面写你申请的第一个域名)
acme.sh --install-cert -d reality.yourdomain1.com --ecc \
--key-file /etc/ssl/private/private.key \
--fullchain-file /etc/ssl/private/fullchain.cer
```

## 4. 编译安装 Nginx 与服务配置

直接复制下方整段命令执行，完成 1.27.3 版本的编译安装：

```bash
# 先更新软件源索引
apt-get update

# 安装编译依赖并进行编译
apt-get install -y gcc g++ libpcre3 libpcre3-dev zlib1g zlib1g-dev openssl libssl-dev wget sudo make curl socat cron && wget https://nginx.org/download/nginx-1.27.3.tar.gz && tar -xvf nginx-1.27.3.tar.gz && cd nginx-1.27.3 && ./configure --prefix=/usr/local/nginx --sbin-path=/usr/sbin/nginx --conf-path=/etc/nginx/nginx.conf --with-http_stub_status_module --with-http_ssl_module --with-http_realip_module --with-http_sub_module --with-stream --with-stream_ssl_module --with-stream_ssl_preread_module --with-http_v2_module && make && make install && cd && mkdir /var/log/nginx

# 上面的报错就使用这个
apt-get install -y gcc g++ libpcre2-dev zlib1g-dev libssl-dev wget sudo make curl socat cron && \
wget "https://nginx.org/download/nginx-1.27.3.tar.gz" && \
tar -xvf nginx-1.27.3.tar.gz && \
cd nginx-1.27.3 && \
./configure --prefix=/usr/local/nginx --sbin-path=/usr/sbin/nginx --conf-path=/etc/nginx/nginx.conf --with-http_stub_status_module --with-http_ssl_module --with-http_realip_module --with-http_sub_module --with-stream --with-stream_ssl_module --with-stream_ssl_preread_module --with-http_v2_module && \
make && \
make install && \
cd .. && \
mkdir -p /var/log/nginx

# 创建 Nginx 的 systemd 服务
cat > /etc/systemd/system/nginx.service << EOF
[Unit]
Description=A high performance web server and a reverse proxy server
Documentation=man:nginx(8)
After=network.target nss-lookup.target

[Service]
Type=forking
PIDFile=/run/nginx.pid
ExecStartPre=/usr/sbin/nginx -t -q -g 'daemon on; master_process on;'
ExecStart=/usr/sbin/nginx -g 'daemon on; master_process on;'
ExecReload=/usr/sbin/nginx -g 'daemon on; master_process on;' -s reload
ExecStop=-/sbin/start-stop-daemon --quiet --stop --retry QUIT/5 --pidfile /run/nginx.pid
TimeoutStopSec=5
KillMode=mixed

[Install]
WantedBy=multi-user.target
EOF

# 重载并设置开机启动
systemctl daemon-reload && systemctl enable nginx.service
```
